Love Matcher API

REST API Documentation • Version 1.0

Authentication

Authorization: Bearer <jwt_token>

All API calls except /register and /ping require these headers.

Rate Limiting

X-RateLimit-Limit: <requests_per_window>
X-RateLimit-Remaining: <requests_remaining>
X-RateLimit-Reset: <window_reset_time>

Endpoints

Registration

POST /register

Create a new user account

Request:

{ "email": "string", "password": "string", "name": "string", "age": number }

Response:

{ "user_id": "string", "token": "string", "status": "active" }

Health Check

GET /ping

Basic health check endpoint

{"status": "ok"}

Profile Management

GET /api/profiles/{user_id}

Retrieve a user profile

POST /api/profiles

Update current user's profile

{ "name": "string", "age": number, "preferences": { "interests": ["string"], "location": "string", "max_age_difference": number } }
DELETE /api/profiles

Delete current user's profile and all associated data

Chat

POST /api/chat

Send a message to another user

{ "to": "user_id", "content": "string" }
GET /api/chat?with={user_id}

Retrieve conversation with another user

{ "messages": [ { "id": "string", "from": "user_id", "content": "string", "timestamp": "ISO-8601" } ], "participants": { "user_id": "name", "other_id": "name" } }

Error Responses

400 Bad Request

{ "error": "Missing required fields" }

401 Unauthorized

{ "error": "Invalid auth token" }

409 Conflict

{ "error": "Email already registered" }

429 Too Many Requests

{ "error": "Rate limit exceeded" }
